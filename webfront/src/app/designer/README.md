# 白U操作计划生成工具

## 功能说明

这个工具用于生成和管理白U操作计划，确保资金经过预处理和Enclave隐私化处理。

## 使用方法

### 1. 访问工具

在浏览器中访问 `/designer` 路径即可使用该工具。

### 2. 管理白U地址

1. 在"白U地址列表"部分，输入地址信息：
   - **地址**: 白U钱包地址
   - **标签**: 可选的地址标签（便于识别）
   - **链**: 选择地址所在的链（TRON/BSC/Ethereum）

2. 点击"添加地址"按钮添加地址

3. 地址会保存在浏览器本地存储中，下次访问时会自动加载

4. 可以点击"删除"按钮移除不需要的地址

### 3. 输入参数

1. **每天需要处理的总量 A**: 输入需要处理的总金额（USDT）

2. **需要处理的地址数 M**: 输入需要处理的目标地址数量

3. **每个地址的需求**: 
   - 点击"添加地址需求"添加一个地址的需求
   - 在输入框中输入每笔交易的金额，按回车或点击"添加"按钮
   - 可以添加多笔交易金额
   - 可以删除不需要的交易金额

### 4. 生成操作计划

1. 确保已输入所有必要参数
2. 点击"生成操作计划"按钮
3. 系统会自动生成满足以下条件的操作计划：
   - 白U总量 >= A × 5
   - 总笔数 >= X × 3（X 是所有地址的总笔数）
   - 存入地址数 >= M × 2

### 5. 查看和导出计划

生成计划后，可以：

1. **查看计划摘要**: 
   - 计划ID
   - 总金额
   - 总笔数
   - 源地址数

2. **查看交易详情**:
   - 每笔交易的ID、金额、源地址、目标地址
   - 交易状态和处理步骤

3. **导出计划**:
   - 点击"导出 JSON"导出为JSON格式
   - 点击"导出 CSV"导出为CSV格式

## 操作计划结构

每笔交易包含以下处理步骤：

1. **预处理 (preprocess)**: 资金存入预处理池（DepositVault）
2. **Enclave隐私化 (enclave)**: 通过Enclave系统进行隐私化处理
3. **转入目标地址 (transfer)**: 最终进入指定的目标地址

## 注意事项

1. **地址管理**: 
   - 白U地址会保存在浏览器本地存储中
   - 清除浏览器数据会丢失保存的地址
   - 建议定期备份地址列表

2. **参数验证**:
   - 确保输入的总量 A 与所有地址需求的总和一致
   - 确保有足够的白U地址（至少 M × 2 个）

3. **计划执行**:
   - 生成的计划仅供参考，实际执行时请根据实际情况调整
   - 建议按照计划顺序执行交易
   - 每笔交易完成后及时更新状态

## 技术说明

- 使用 Next.js App Router
- 数据保存在浏览器 localStorage
- 支持多链（TRON、BSC、Ethereum）
- 响应式设计，支持移动端和桌面端
